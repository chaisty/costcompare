## Notes

**CI workflow.** Three parallel jobs: `web` (npm typecheck + lint + vitest), `edge-functions` (Deno typecheck + tests on `supabase/functions/submit/`), and `db` (Supabase CLI starts a local stack and runs the pgTAP suite). All run on `push` to main and on PRs. The `db` job uses `supabase/setup-cli@v1` and the `edge-functions` job uses `denoland/setup-deno@v2` — both official actions, no manual binary download needed in CI. Real test will be the first PR; no PRs in this session to validate, but the workflow YAML is straightforward.

**Taxonomy mapping is keyword-based.** NPPES taxonomy strings cluster cleanly into our small `facility_type` enum: anything containing "hospital" → `hospital` (deliberately wins over "ambulatory surg" so "Surgical Hospital" maps right), "medical center" → `medical_center`, "ambulatory surg|surgical center" → `asc`, "clinic|practice|health center" → `clinic`, otherwise → `other`. 9 unit tests in `upsert.test.ts` cover the precedence + edge cases (FQHCs, surgical hospitals, case insensitivity, null fallthrough). The mapping is conservative: when in doubt it falls through to `other`, never to a more specific value.

**Medicare-certified badge: small but meaningful.** The `search_rates` RPC now exposes `facility_external_id`. When set, the row was sourced (or cross-validated) against the CMS POS file, which means the facility is Medicare-enrolled. The badge is a single span with a tooltip explaining the source; CTSS-only rows render no badge. Two new Vitest assertions cover both paths. Migration was a drop+recreate of the function (Postgres can't widen the returns shape via ALTER); column allowlist updated in the test along with it.

**UTC-pinned date formatter.** `Intl.DateTimeFormat('en-US', { timeZone: 'UTC', ... })` instead of `new Date(...).toLocaleDateString()`. The viewer's local timezone no longer shifts the date. Was a small but persistent test smell since #7.

**State filter copy clarification.** Two-line change. The state filter matches `facility.state OR provider.practice_state` (since #13's RPC change), so the rendered hint now says so. Avoids the "I picked CA but my Texas-practicing-Cali-licensed doctor disappeared" surprise.
